ON DELETE CASCADE se o registro pai for deletado,o filho também será.

ON UPDATE CASCADE se a primary key do registro pai for alterada, isso será refletido no registro filho.

RESTRICT significa que qualquer tentativa de apagar ou atualizar o registro pai vai falhar lançando um erro.

NO ACTION Sem ações. Significa que o MySQL não vai realizar nenhuma ação por você. Porém, você não poderá apagar ou atualizar a primary key do registro pai, nem conseguirá alterar a foreing key do registro filho caso a primary key do registro pai ainda não tiver sido atualizada.

SET NULL - Se você apagar ou atualizar a primary key do registro pai, a foreing key do registro filho será configurada para NULL.
